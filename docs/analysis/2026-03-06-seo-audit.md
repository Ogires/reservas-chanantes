# SEO Audit - Reservas Chanantes

**Fecha**: 2026-03-06
**Sitio**: Reservas Chanantes - plataforma SaaS multi-tenant de reservas para pequenos negocios
**Stack**: Next.js 16.1.6 (App Router), React 19, TypeScript, Supabase, Stripe, Vercel
**Idioma HTML**: `lang="es"` (contenido mixto ES/EN)

---

## Resumen Ejecutivo

**Estado general del SEO: BAJO** - La aplicacion tiene solo los metadatos basicos. Le faltan casi todos los artefactos estandar que los motores de busqueda necesitan para rastrear, indexar y mostrar resultados enriquecidos.

### Top 5 Problemas Prioritarios
1. Sin `robots.txt` - los crawlers no tienen directivas
2. Sin sitemap XML - el descubrimiento de paginas esta limitado
3. Sin OpenGraph / Twitter cards - cero optimizacion para compartir en redes
4. Sin datos estructurados (JSON-LD) - sin posibilidad de rich snippets
5. Sin URLs canonicas - riesgo de contenido duplicado en paginas de tenants

### Quick Wins Identificados
- Crear `robots.ts` (5 min)
- Crear `sitemap.ts` (15 min)
- Anadir OpenGraph al root layout y paginas `[slug]` (15 min)
- Anadir URL canonica via `metadataBase` (5 min)
- Anadir `generateStaticParams()` a `[slug]/page.tsx` (10 min)

---

## 1. Crawlability e Indexacion

### 1.1 robots.txt - FALTABA
- **Impacto**: ALTO
- **Problema**: No existia `robots.txt`. Los crawlers rastreaban todo por defecto, incluyendo `/admin/*`, `/my/*`, `/api/*` que NO deberian indexarse.
- **Solucion**: Crear `src/app/robots.ts` usando la API de Metadata de Next.js:
  - Allow: `/`, `/[slug]`
  - Disallow: `/admin`, `/my`, `/api`
  - Incluir referencia al sitemap
- **Archivo**: `src/app/robots.ts` (nuevo)

### 1.2 Sitemap XML - FALTABA
- **Impacto**: ALTO
- **Problema**: No existia sitemap. Google descubria paginas de tenants solo por crawling.
- **Solucion**: Crear `src/app/sitemap.ts` que genera dinamicamente URLs para:
  - Homepage `/`
  - Todas las paginas de reserva activas `/{slug}`
- Consulta la tabla `tenants` para obtener tenants activos
- **Archivo**: `src/app/sitemap.ts` (nuevo)

### 1.3 URLs Canonicas - FALTABAN
- **Impacto**: ALTO
- **Problema**: Sin `metadataBase` en el root layout. Sin URLs canonicas generadas. Las paginas multi-tenant arriesgan senales de contenido duplicado.
- **Solucion**: Anadir `metadataBase` al metadata del root layout. Next.js auto-genera URLs canonicas y resuelve URLs de imagenes OG.
- **Archivo**: `src/app/layout.tsx` (editar)

### 1.4 Paginas Admin/Portal Indexables
- **Impacto**: MEDIO
- **Problema**: Las paginas `/admin/*` y `/my/*` no tenian directiva `noindex`. Incluso con auth, las paginas de login pueden rastrearse e indexarse.
- **Solucion**: Anadir `robots: { index: false, follow: false }` a los layouts de admin y customer portal.
- **Archivos**: `src/app/admin/layout.tsx`, `src/app/my/layout.tsx` (editar)

### 1.5 Sin `not-found.tsx` ni `error.tsx`
- **Impacto**: BAJO
- **Problema**: Sin pagina 404 personalizada. El 404 default de Next.js no tiene valor SEO ni enlaces internos para los crawlers.
- **Solucion**: Crear `src/app/not-found.tsx` con enlaces al homepage.
- **Archivo**: `src/app/not-found.tsx` (nuevo)

## 2. SEO On-Page

### 2.1 Title Tags - BASICO
- **Impacto**: MEDIO
- **Problema**:
  - El titulo de la homepage es generico: "Reservas Chanantes" - sin keywords
  - Paginas de tenant: "Book with {name}" - aceptable pero mejorable
  - Sin template de titulo para branding consistente
- **Solucion**:
  - Root layout: Usar `title: { default: 'Reservas Chanantes - Reservas Online para Negocios', template: '%s | Reservas Chanantes' }`
  - Paginas de tenant: `"Reserva cita en ${tenant.name} - Reservas Online"`

### 2.2 Meta Descriptions - BASICO
- **Impacto**: MEDIO
- **Problema**:
  - Homepage: "Online booking platform for small businesses" - en ingles, generico
  - Paginas de tenant: "Schedule your appointment online with {name}." - en ingles, pobre
- **Solucion**: Escribir descripciones en espanol, ricas en keywords:
  - Homepage: "Plataforma de reservas online para pequenos negocios. Crea tu pagina de reservas en minutos y permite a tus clientes reservar citas 24/7."
  - Paginas de tenant: "Reserva tu cita online en ${tenant.name}. Elige servicio, dia y hora disponible. Confirmacion inmediata."

### 2.3 Estructura de Headings - OK
- **Impacto**: BAJO
- **Problema**: La homepage y paginas de tenant tienen H1 correcto. La seccion de features usa H3 (salta H2).
- **Solucion**: Envolver la seccion de features en H2 para que la jerarquia sea H1 > H2 > H3.

### 2.4 Contenido en Idioma Mixto
- **Impacto**: MEDIO
- **Problema**: HTML `lang="es"` pero el contenido esta mayormente en ingles ("Online bookings, made simple", "Get Started Free"). Google puede confundirse sobre el idioma/mercado objetivo.
- **Solucion**: Traducir contenido publico a espanol. El contenido admin/portal puede permanecer en ingles.

## 3. Metadata Social y de Compartir

### 3.1 OpenGraph Metadata - FALTABA
- **Impacto**: ALTO
- **Problema**: Sin tags OpenGraph. Al compartir en redes sociales, no aparece imagen, titulo ni descripcion.
- **Solucion**: Anadir datos OG al root layout y por-tenant en `[slug]/page.tsx`.

### 3.2 Twitter Card Metadata - FALTABA
- **Impacto**: MEDIO
- **Solucion**: Anadir `twitter: { card: 'summary_large_image' }` al metadata del root layout.

---

## 4. Datos Estructurados (JSON-LD)

### 4.1 Sin Datos Estructurados - FALTABA
- **Impacto**: ALTO
- **Problema**: No habia markup JSON-LD en ninguna parte. Oportunidad perdida de rich results.
- **Solucion**: Anadir JSON-LD para:
  - **Homepage**: Schema `SoftwareApplication`
  - **Paginas de tenant**: Schema `LocalBusiness` + `Service` con ofertas y precios

---

## 5. Rendimiento Tecnico

### 5.1 `generateStaticParams()` - FALTABA
- **Impacto**: MEDIO
- **Problema**: `[slug]/page.tsx` no tenia `generateStaticParams()`. Todas las paginas de tenant se renderizan dinamicamente en cada peticion, aumentando el TTFB.
- **Solucion**: Anadir `generateStaticParams()` para pre-renderizar paginas de tenant en build time.

### 5.2 Optimizacion de Imagenes - NO APLICA
- Sin imagenes de usuario en paginas publicas (solo iconos SVG decorativos).

### 5.3 Carga de Fuentes - OK
- Google Fonts via `next/font` con `display: swap` (default de Next.js). Bueno para CLS.

### 5.4 next.config.ts - VACIO
- **Impacto**: BAJO
- **Solucion**: Anadir security headers (X-Frame-Options, CSP, etc.) para mejor postura de confianza.

---

## 6. PWA Manifest

### 6.1 Manifest Incompleto
- **Impacto**: BAJO
- **Problema**: `manifest.json` no tiene `start_url`, `description`, ni `scope`.
- **Solucion**: Anadir campos faltantes.

---

## 7. Pagina de Exito de Reserva

### 7.1 Sin noindex en Pagina Transaccional
- **Impacto**: BAJO
- **Problema**: `[slug]/booking-success/page.tsx` no tiene metadata y podria indexarse. Estas paginas de confirmacion no tienen valor SEO.
- **Solucion**: Anadir `robots: { index: false }`.
